4. 4xx(클라이언트 에러)와 5xx (서버 에러)의 차이점 메모

- 4xx (클라이언트 에러) 종류

  1. 400 Bad Request - 요청 구문이 잘못되었거나 서버가 이해할 수 없는 요청일 때 발생 (URL 오타, 쿠키/캐시 문제 등)

  2. 401 Unauthorized (미승인): 요청을 하려면 인증이 필요함을 의미하고 로그인 정보가 없거나 잘못된 경우 발생됨

  3. 403 Forbidden (금지됨) : 서버는 요청을 알았지만, 권한이 없어 접근이 거부한 경우

  4. 404 Not Found(찾을 수 없음) : 요청한 리소스(페이지, 파일)가 서버에 존재하지 않을 때 발생함. URL 오타나 링크 깨짐이 원인일 수 있다.

  5. 408 Request Timeout(요청 시간 초과) : 서버가 일정 시간 동안 요청을 기다렸으나 응답이 없을 때 발생함.

- 5xx (서버 에러)

  1. 500 Internal Server Error(내부 서버 오류) : 서버가 요청을 처리하는 중 예상치 못한 상황에 처했을 때 발생하는 일반적인 오류.

  2. 503 Service Unavailable (서비스 사용 불가) : 서버가 일시적으로 과부화되거나 유지보수 중이어서 요청을 처리할 수 없을 떄 나타냄.

  3. 504 GateWay Timeout (게이트웨이 시간 초과) : 게이트웨이나 프록시 서버가 다른 서버로부터 응답을 받는데 너무 오래 걸릴 때 발생함.

  4. 502 Bad GateWay (잘못된 게이트웨이) : 게이트웨이 역할을 하는 서버가 업스트림 서버로부터 잘못된 응답을 받았을 때 발생함.
