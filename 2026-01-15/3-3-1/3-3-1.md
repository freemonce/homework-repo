<h1>공개 API 응답 분석하기</h1>

1. 브라우저에서 공개 API URL을 직접 접속해보세요 -https://dog.ceo/api/breeds/image/random

2. 응답 데이터의 JSON 구조를 분석하고 각 필드의 의미를 작성하세요
   - {"message":"https:\/\/images.dog.ceo\/breeds\/ridgeback-rhodesian\/n02087394_9126.jpg","status":"success"}

- message : API가 전달하고자 하는 핵심 데이터, https:\/\/images.dog.ceo\/breeds\/
  랜덤하게 뽑힌 강아지 사진의 실제 주소
- status : 문제없이 성공적으로 사진을 찾아냈다.

3. Request와 Response의 차이를 실제 예시와 함께 설명하세요

- Request (요청):
  사용자가 브라우저 주소창 : https://dog.ceo/api/breeds/image/random
  입력 및 엔터 누름
  (서버에 데이터 보내줘)

- Response (응답) : 서버가 로직에 따라 데이터베이스에서 사진 한 장을 골라 {"message": "...", "status": "success"}라는 텍스트 뭉치를 화면에 띄워줌.

4. API 키가 필요한 이유와 보안상 주의점을 정리하세요

- API 키가 필요한 이유

  - 누가 쓰는 지 확인함
  - 과도한 사용방지
  - 권한 관리

- 보안상 주의점
  - 공개 금지
  - 환경 변수 활용
